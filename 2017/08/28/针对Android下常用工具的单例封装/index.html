发表于 2017-08-28 | 分类于 Android | 浏览 次

单例模式

单例优点不言而喻，它能够保证在整个类中，实际的实例化的对象只有一个，大大节约了内存资源，设计思路也非常简单：一个类能返回对象一个引用(永远是同一个)和一个获得该实例的方法（必须是静态方法，通常使用getInstance这个名称）
Java中常见的单例模式有两种“饿汉模式”和“懒汉模式”，不常见的就不说了。

懒汉模式

实际在安卓项目中，手机内存比起旧式智能机大大增加，一般的封装模式是采用懒汉式，“用空间去换取时间”设计思路。

Gson

Gson作为Google官方的将json和model互相转换的工具也需要封装为一个单例模式,因为Gson gson = new Gson();会消耗大量资源，所以共享单例才是王道,封装起来也比较简单方便，看个人习惯。
代码实例：

使用方法：

Model model = GsonUtil.getGsonInstance().fromJson(data, Model.class);
